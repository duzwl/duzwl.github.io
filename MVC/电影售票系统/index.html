电影售票系统技术详解

  • Java
  • MVC
MVC

这学期在学习了Spring框架之后，课程作业是开发一个电影售票系统，在团队开发过程中，我主要参与后台开发，下面是系统开发过程中的一些要点与总结。

互联网售票软件是比较常见的软件系统。通常由多个零售系统和多个供给系统系统。 机票、酒店房间、电影票似乎是风马牛大相关的系统，但它们之间存在极其相似的业务模型。 以电影票为例，格瓦拉、蜘蛛网、腾讯等等都做类似的电影票分销、推广业务，但票是由各大院线排期提供的。 分销-院线-影院形成了一个完整的生态体系。本次大作业实现一个简单的互联网电影售票系统。服务端采用Java语言， Spring 4 作为web开发技术框架， jesery 作为 RESTful 服务提供框架。

在这次大作业中，个人贡献主要包括两个方面：数据库数据录入工作、学习Spring框架并参与Java后台开发相关工作。以下是本文的行文结构：

电影售票系统简介

基本功能

基本功能包括：1. 电影资讯；2. 购票；3. 用户信息；4. 收藏；5. 评论

用户使用该可以实现查看和搜索正在热映和即将上映的电影资讯，对感兴趣的电影可以进行收藏。想要观看某部电影，可以进行下单购票操作。对于已经放映的电影订单，用户可以对该订单对应的电影进行评分评价的操作。另外，用户还可以更新自己的个人信息。下面是整个应用的界面逻辑概要图，方便对应用有个直观的了解：

  • 查看电影用例

    用户可以通过查看资讯来查看电影，也可以通过直接搜索来查看电影。电影资讯包括两种类型：正在热映和即将上映。除了以上两种方式外，在应用首页也会推荐一些热门电影供用户查看。不管是以何种方式查看电影，都可以点击进入详情页面查看某部电影的详细信息。

  • 下单购票

    选择想要观看的电影之后，可以进行购票操作。购票时需要选择该电影放映的场次和座位信息。基本信息选择完成之后就可以创建订单了，创建订单的时候，如果用户还有优惠券，那么可以选择使用优惠券。订单创建完成后可以对订单进行支付，支付订单需要输入支付密码。最后，订单完成支付后用户可以看到订单的详细信息，系统还会随机送给用户优惠券。

数据库数据录入

项目开始阶段，数据库表格的数据是手工录入的，数据量不大。主要包括以下表格，这些表格在spring架构中有对应的类来读取相关的数据：

  • movie表：条目为(id, name, type, stars)，分别表示电影的id、电影名、电影所属类型、电影星级
  • cinema表：条目为(id, name, address, stars, info, max_discount, town)，分别表示电影院的id，名字、地址、星级、关于影院的额外信息、影院折扣
  • movieItem表：条目为(id, movie, cinema, seat, price, duration)，分别表示电影场次的id、电影在movie表中的id、在cinema表中的id、票价、作为总数、电影时长
  • city表：条目为(id, name, province)，分别表示城市的id、名字、所在省份的id
  • province表：条目为(id, name)，分别表示省份的id、名字
  • town表：条目为(id, name, city)，分别表示县的id、名字、所在城市的id

表格中的数据由相应的DAO对象来获取，然后再返回给前端；唯一的不足之处是，本次作业中的数据库数据量有点少，这个在往后还需要编写相关的爬虫算法扒取商业网站上的电影数据。

Spring MVC框架在本次售票系统中的具体应用

下面详细介绍电影售票系统中五个主要的包的代码功能：

  • teamid9527.moviebooking.dao：各种数据获取类，包括CinemaDao, CityDao, CustomerDao, MovieDao, MovieItemDao, ProvinceDao, ReservationDao, TownDao等，其功能是获取各种相关的数据，比如查找所有上映的电影，查找用户的名字，更新用户在数据库中的相关数据等等
  • teamid9527.moviebooking.entities：基本的实体，包括Cinema, City, Customer, Movie, MovieItem, Province, Reservation, Town等，实体包含各自独有的属性，比如名字，id，地址等等，这里不累述
  • teamid9527.moviebooking.exception：包括DeleteException, InsertException, LoginException, QueryException, RegisterException, UpdateException等，用来处理各种数据处理异常或者逻辑异常
  • teamid9527.moviebooking.service：包括CustomerService（用户登录、注册、更新用户数据等）、MovieItemService（通过电影院查询上映的电影）、ReservationService（查询用户的订单、添加订单、删除订单、更新订单等）
  • teamid9527.moviebooking.handler：包括CustomerHandler（对于前端的不同路由请求做出响应，路由功能见下文）、MyInterceptor

由此可见，Spring MVC在本系统中的使用如下：对于不同的前端路由请求，CustomerHandler接收到路由请求后，调用不同的service，比如CustomerService、ReservationService等，这些serivce又调用底层的entities和DAO获得数据，当然若处理过程中出现异常，需要用我们自己编写的exceptin类来作出相关处理。这里，前端属于V即view，CustomerHandler属于C即controller，entities和DAO等属于M即model。

以下是各种路由处理：

  • 登录
    • /：使用GET，前往登录页面
    • gotoLogin：使用GET，前往登录页面
    • login：使用POST，提交表单进行用户登录，若成功前往用户详情页面，否则返回登录页面
  • 注册
    • gotoLRegister：使用GET，前往注册页面
    • register：使用POST，提交表单进行用户注册，若成功前往用户详情页面，否则返回登录页面
  • 用户详情
    • update：使用POST，提交表单进行用户更新，若成功前往用户详情页面，否则返回登录页面
    • backToInfo：使用GET，返回用户详情页面，若成功前往用户详情页面，否则返回登录页面
  • 查询用户预订
    • reservation：使用GET，查询用户订单，若成功前往用户订单页面，否则返回登录页面
  • 修改用户预订
    • MovieItem/{id}：使用DELETE，删除用户订单中的id电影场次，若成功返回用户订单页面，否则返回登录页面
    • /MovieItem/{id}：使用PUT，向用户订单中添加id电影场次，若成功返回选票页面，否则返回登录页面
  • 查询影院
    • queryCinema：使用GET，查询所有的影院
  • 查询电影
    • queryCinema：使用GET，查询所有的电影
  • 查询电影场次
    • queryMovieItemsByCinema?id={id}：使用GET，查询id电影院放映的电影场次，返回电影场次
    • queryMovieItemsByMovie？id={id}：使用GET，查询id电影上映的电影场次，返回电影场次
